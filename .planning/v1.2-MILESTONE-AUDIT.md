### Integration Notes (Non-Blocking)

1. **REST revoke accepts UUID only** — `DELETE /keys/:id` requires full UUID; CLI supports 8-char display_id lookup via `find_by_display_id`. Asymmetry is by design; no requirement mandates display_id on REST.
2. **auth_mode cosmetic inaccuracy** — `ApiError::Unauthorized` always emits `"auth_mode": "active"` in JSON body, including the DB-error fallback path where "active" is semantically inaccurate. Cosmetic only.
