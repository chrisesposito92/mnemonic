### Integration Gap: recall CLI Bypass

`run_recall()` at `src/cli.rs:455` takes `Arc<tokio_rusqlite::Connection>` and executes raw SQLite SQL in `cmd_list_memories` (line 578) and `cmd_get_memory` (line 668). On non-SQLite deployments, `mnemonic recall` returns empty/stale data from the local SQLite file.

- **Root cause:** `main.rs:38` calls `init_db()` (not `init_db_and_embedding()`) for the recall path
- **Fix:** Change `run_recall` to accept `MemoryService`, call `init_db_and_embedding()` for recall
- **Severity:** Non-blocking — recall is a v1.3 feature, not in v1.4 requirements scope
- **Affected paths:** STOR-03 (recall bypasses MemoryService), QDRT-04/PGVR-04 (agent isolation not exercised via recall CLI)
