    - Greedy clustering with 3 vectors where A-B similarity > threshold but A-C and B-C below: produces 1 cluster [A,B], C is singleton (not clustered)
    - Greedy clustering with first-match: if A-B=0.95 and A-C=0.90 (both above threshold), A joins cluster with B first, then C joins that same cluster via (Some(id), None) arm
    - Greedy clustering with both-assigned: if A-B and C-D are clustered, and B-C is above threshold, skip (both already assigned)
    - Empty candidates produce 0 clusters, empty response

**Pure helper functions (non-pub, synchronous, unit-testable):**

1. `fn cosine_similarity(a: &[f32], b: &[f32]) -> f32` — dot product (vectors are already L2-normalized by EmbeddingEngine)

2. `fn compute_pairs(candidates: &[CandidateMemory], threshold: f32) -> Vec<SimilarityPair>` — iterate all i<j pairs, filter by >= threshold, collect into Vec, sort by descending similarity

3. `fn cluster_candidates(pairs: &[SimilarityPair], num_candidates: usize) -> Vec<Vec<usize>>` — greedy first-match assignment:
   - `cluster_id: Vec<Option<usize>>` initialized to `vec![None; num_candidates]`
   - For each pair in order (already sorted desc): match on (cluster_id[i], cluster_id[j]):
     - (None, None) -> new cluster with both
     - (Some(id), None) -> add j to existing cluster id
     - (None, Some(id)) -> add i to existing cluster id
     - (Some(_), Some(_)) -> skip
   - Return only clusters with 2+ members

4. `fn tier1_concat(memories: &[&CandidateMemory]) -> String` — sort by created_at ascending, join content with "\n"

5. `fn union_tags(memories: &[&CandidateMemory]) -> Vec<String>` — iterate all tags, use HashSet for dedup, preserve insertion order

6. `fn earliest_created_at(memories: &[&CandidateMemory]) -> String` — min by created_at string comparison (ISO 8601 sorts lexicographically)

**Async methods on CompactionService:**

7. `async fn fetch_candidates(&self, agent_id: &str, max_candidates: u32) -> Result<(Vec<CandidateMemory>, bool), ApiError>` — returns (candidates, truncated)
   - SQL: `SELECT m.id, m.content, m.tags, m.created_at, v.embedding FROM memories m JOIN vec_memories v ON v.memory_id = m.id WHERE m.agent_id = ?1 ORDER BY m.created_at DESC LIMIT ?2`
   - Parameter ?2 = max_candidates + 1 (fetch one extra to detect truncation)
   - If result.len() > max_candidates, truncated=true, truncate to max_candidates
   - Tags: `row.get::<_, String>(2)?` then `serde_json::from_str(&tags_str).unwrap_or_default()`
   - Embedding bytes: `row.get::<_, Vec<u8>>(4)?` then unsafe reinterpret: `unsafe { std::slice::from_raw_parts(bytes.as_ptr() as *const f32, bytes.len() / 4).to_vec() }`
   - Add `// SAFETY: sqlite-vec stores 384 aligned f32 values as 1536 bytes (IEEE 754 little-endian)`

8. `async fn synthesize_content(&self, cluster_memories: &[&CandidateMemory]) -> String`
   - If self.summarization is Some: call engine.summarize() with texts collected from cluster_memories
   - On Ok(summary): return summary
   - On Err(e): `tracing::warn!(error = %e, "LLM summarization failed, falling back to Tier 1");` then fall through to Tier 1
   - If self.summarization is None or LLM failed: return `tier1_concat(cluster_memories)`

9. `pub async fn compact(&self, req: CompactRequest) -> Result<CompactResponse, ApiError>` — the main pipeline:
   - Extract defaults: `let threshold = req.threshold.unwrap_or(0.85);`
   - `let max_candidates = req.max_candidates.unwrap_or(100);`
   - `let dry_run = req.dry_run.unwrap_or(false);`
   - Create compact_runs record (status='running') via db.call
   - fetch_candidates
   - If candidates is empty: update compact_runs to 'completed' with 0 counts, return empty response
   - compute_pairs + cluster_candidates
   - For each cluster: gather &CandidateMemory refs, synthesize_content, union_tags, earliest_created_at
   - Re-embed merged content via self.embedding.embed(&merged_content)
   - If NOT dry_run: atomic write transaction inside a single db.call:
     - `let tx = c.transaction()?;`
     - INSERT INTO memories (id, content, agent_id, session_id, tags, embedding_model, created_at, source_ids) VALUES (...)
     - INSERT INTO vec_memories (memory_id, embedding) VALUES (...)
     - DELETE FROM vec_memories WHERE memory_id = ? (for each source_id)
     - DELETE FROM memories WHERE id = ? (for each source_id)
     - `tx.commit()?;`
   - Build ClusterMapping for each cluster: source_ids = cluster source IDs, new_id = Some(new_id) if not dry_run, None if dry_run
   - Update compact_runs to 'completed' with counts
   - Return CompactResponse
   - On any error in the write transaction: update compact_runs to 'failed' via separate db.call, then propagate error
