FROM memories
WHERE agent_id = $1
ORDER BY created_at DESC

```

3. Bind `agent_id` as `$1`, `fetch_limit` as `$2`.

4. Execute with `.fetch_all(&self.pool).await.map_err(map_db_err)?`.

let candidates: Vec<CandidateRecord> = rows.into_iter()
    .take(max_candidates as usize)

        let id: String = row.try_get("id").map_err(map_db_err)?;
        let content: String = row.try_get("content").map_err(map_db_err)?;

    })
    .collect::<Result<Vec<_>, ApiError>>()?;
```

7. Return `Ok((candidates, truncated))`.

IMPORTANT: The `embedding` column is retrieved as `pgvector::Vector` via `row.try_get::<Vector, _>("embedding")`. Convert to `Vec<f32>` using `.into()` (pgvector implements `From<Vector> for Vec<f32>`).

**write_compaction_result() implementation (per D-13, D-14):**

Uses a Postgres transaction for full atomicity — the key advantage over Qdrant.

1. Begin transaction: `let mut tx = self.pool.begin().await.map_err(map_db_err)?;`

2. INSERT the merged memory within the transaction:
```sql
INSERT INTO memories (id, content, agent_id, session_id, tags, embedding_model, created_at, embedding)
VALUES ($1, $2, $3, $4, $5, $6, $7::timestamptz, $8::vector)
```
Bind values:
- `$1` = `&req.new_id`
- `$2` = `&req.content`
- `$3` = `&req.agent_id`
- `$4` = `""` (empty string for session_id — merged compaction memories have no session, per pitfall 6 from RESEARCH.md)
- `$5` = `&req.tags[..]` (as `&[String]` for TEXT[])
- `$6` = `&req.embedding_model`
- `$7` = `&req.created_at` (ISO 8601 string cast to timestamptz via `$7::timestamptz`)
- `$8` = `Vector::from(req.embedding.clone())` (pgvector Vector)

Execute with `.execute(&mut *tx).await.map_err(map_db_err)?` — CRITICAL: use `&mut *tx` (DerefMut), NOT `&tx`. See Research pitfall 3.

3. DELETE source memories within the same transaction:
```sql
DELETE FROM memories WHERE id = ANY($1)
```
Bind `&req.source_ids[..]` (as `&[String]` which sqlx encodes as TEXT[]).
Execute with `.execute(&mut *tx).await.map_err(map_db_err)?`.

4. Commit: `tx.commit().await.map_err(map_db_err)?;`

If any step fails, sqlx automatically rolls back the transaction when `tx` is dropped (per Research "Don't Hand-Roll" section). This guarantees atomicity per D-14.

5. Return the Memory:
```rust
Ok(Memory {
    id: req.new_id,
    content: req.content,
    agent_id: req.agent_id,
    session_id: "".to_string(),
    tags: req.tags,
    embedding_model: req.embedding_model,
    created_at: req.created_at,
    updated_at: None,
})
```

Do NOT fetch the row back after insert — construct the Memory directly from the request fields (same pattern as QdrantBackend's write_compaction_result).
