**7. Helper function — row_to_memory():**
Extract a `Memory` from an `sqlx::postgres::PgRow` using `sqlx::Row::try_get()`:
```rust
use sqlx::Row;

fn row_to_memory(row: &sqlx::postgres::PgRow) -> Result<Memory, ApiError> {
    Ok(Memory {
        id: row.try_get("id").map_err(map_db_err)?,
        content: row.try_get("content").map_err(map_db_err)?,
        agent_id: row.try_get("agent_id").map_err(map_db_err)?,
        session_id: row.try_get("session_id").map_err(map_db_err)?,
        tags: row.try_get::<Vec<String>, _>("tags").map_err(map_db_err)?,
        embedding_model: row.try_get("embedding_model").map_err(map_db_err)?,
        created_at: row.try_get::<String, _>("created_at").map_err(map_db_err)?,
        updated_at: row.try_get::<Option<String>, _>("updated_at").map_err(map_db_err)?,
    })
}
```
IMPORTANT: The `created_at` and `updated_at` columns are TIMESTAMPTZ in the database. They must be retrieved as strings. In all SELECT queries, use `TO_CHAR(created_at AT TIME ZONE 'UTC', 'YYYY-MM-DD"T"HH24:MI:SS"Z"') AS created_at` and similarly for `updated_at` (with `TO_CHAR(updated_at AT TIME ZONE 'UTC', 'YYYY-MM-DD"T"HH24:MI:SS"Z"') AS updated_at`). This casts the TIMESTAMPTZ to a TEXT column in the result set, which sqlx decodes as `String` / `Option<String>` without needing chrono. See Research pitfall 5.

**8. StorageBackend trait impl:**

`store()` (per D-01, D-22, D-23):
- Build SQL: `INSERT INTO memories (id, content, agent_id, session_id, tags, embedding_model, embedding) VALUES ($1, $2, $3, $4, $5, $6, $7::vector)`
- Bind: `&req.id`, `&req.content`, `&req.agent_id`, `&req.session_id`, `&req.tags[..]` (as `&[String]` for TEXT[]), `&req.embedding_model`, `Vector::from(req.embedding.clone())`
- Execute via `sqlx::query(...).bind(...).execute(&self.pool).await.map_err(map_db_err)?`
- Then SELECT the inserted row back using `get_by_id(&req.id)` and unwrap the Option (it must exist since we just inserted it). Alternatively, use a second query: `SELECT id, content, agent_id, session_id, tags, embedding_model, TO_CHAR(created_at AT TIME ZONE 'UTC', 'YYYY-MM-DD"T"HH24:MI:SS"Z"') AS created_at, TO_CHAR(updated_at AT TIME ZONE 'UTC', 'YYYY-MM-DD"T"HH24:MI:SS"Z"') AS updated_at FROM memories WHERE id = $1` with `.fetch_one(&self.pool)`, then call `row_to_memory(&row)`.
- Return the Memory.

`get_by_id()`:
- SQL: `SELECT id, content, agent_id, session_id, tags, embedding_model, TO_CHAR(created_at AT TIME ZONE 'UTC', 'YYYY-MM-DD"T"HH24:MI:SS"Z"') AS created_at, TO_CHAR(updated_at AT TIME ZONE 'UTC', 'YYYY-MM-DD"T"HH24:MI:SS"Z"') AS updated_at FROM memories WHERE id = $1`
- Use `.fetch_optional(&self.pool).await.map_err(map_db_err)?`
- Map `Some(row)` to `Some(row_to_memory(&row)?)`, None to `Ok(None)`

`delete()`:
- First call `self.get_by_id(id).await?` and return `ApiError::NotFound` if None (same fetch-then-delete pattern as QdrantBackend)
- Then execute `DELETE FROM memories WHERE id = $1` with `.execute(&self.pool).await.map_err(map_db_err)?`
- Return the memory fetched before deletion
