  <action>
1. Create `src/config.rs` with:
   - `#[derive(Debug, Clone, Deserialize, Serialize)]` struct `Config` with fields:
     - `pub port: u16` (default 8080)
     - `pub db_path: String` (default "./mnemonic.db")
     - `pub embedding_provider: String` (default "local")
   - Implement `Default for Config` with the above defaults
   - `pub fn load_config() -> Result<Config, crate::error::ConfigError>` that:
     1. Reads `MNEMONIC_CONFIG_PATH` env var; falls back to "mnemonic.toml" if unset
     2. Creates `Figment::from(Serialized::defaults(Config::default()))`
     3. Merges `Toml::file(&toml_path)` (uses `Toml::file`, NOT `Toml::file_exact` — missing file is silently ignored)
     4. Merges `Env::prefixed("MNEMONIC_")` (env vars win over TOML)
     5. Calls `.extract::<Config>()` and maps errors to `ConfigError::Load(format!("{}", e))`

2. Add `#[cfg(test)] mod tests` at the bottom of config.rs with tests using figment's `Jail` test helper for env isolation:
   - `test_config_defaults`: no env, no TOML -> port 8080, db_path "./mnemonic.db", embedding_provider "local"
   - `test_config_env_override`: set MNEMONIC_PORT=9090 -> port 9090
   - `test_config_toml_override`: write mnemonic.toml with `port = 7070` -> port 7070
   - `test_config_env_beats_toml`: set MNEMONIC_PORT=9090 AND write toml with port=7070 -> port 9090
   - `test_config_missing_toml_ok`: no toml file -> defaults returned, no error

   Use `figment::Jail::expect_with(|jail| { ... })` for each test to get an isolated temp directory and env.
   Inside the jail, use `jail.set_env("MNEMONIC_PORT", "9090")` and `jail.create_file("mnemonic.toml", "port = 7070\n")` as needed.
   IMPORTANT: figment Jail changes CWD to its temp dir, so Toml::file("mnemonic.toml") picks up jail-created files automatically.

Note: Since this is the first Rust file being created and there is no main.rs yet, add a temporary `fn main() {}` stub in a `src/main.rs` to allow `cargo test` to compile the crate. The stub just needs: `mod config; mod error; fn main() {}`. This will be replaced in Plan 02.
