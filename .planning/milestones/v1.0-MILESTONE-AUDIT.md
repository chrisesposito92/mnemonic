### E2E Flows (5/5 complete)

1. **Store a memory** — POST /memories -> embed -> atomic dual-table insert -> 201 + Memory JSON
2. **Semantic search** — GET /memories/search -> embed query -> CTE KNN over-fetch -> JOIN agent filter -> ranked results
3. **List memories** — GET /memories -> IS NULL OR filter pattern -> COUNT + SELECT -> paginated response
4. **Delete a memory** — DELETE /memories/:id -> transactional delete both tables -> 200 or 404
5. **Server startup** — register_sqlite_vec -> init_tracing -> load_config -> validate_config -> db::open -> engine selection -> MemoryService::new -> serve

None. INT-01 and INT-02 from previous audit are both closed by Phase 5.
