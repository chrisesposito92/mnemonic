**Add create_key_handler (D-15):**
```rust
/// POST /keys — creates a new API key and returns the raw token (shown once).
async fn create_key_handler(
    State(state): State<AppState>,
    Json(body): Json<CreateKeyRequest>,
) -> Result<(axum::http::StatusCode, Json<serde_json::Value>), ApiError> {
    let (api_key, raw_token) = state.key_service.create(body.name, body.agent_id).await
        .map_err(|e| ApiError::Internal(crate::error::MnemonicError::Db(e)))?;
    // CRITICAL: Do NOT log raw_token (PITFALLS.md Auth Pitfall 6)
    tracing::info!(key_id = %api_key.id, agent_id = ?api_key.agent_id, "API key created");
    Ok((
        axum::http::StatusCode::CREATED,
        Json(serde_json::json!({
            "key": {
                "id": api_key.id,
                "name": api_key.name,
                "display_id": api_key.display_id,
                "agent_id": api_key.agent_id,
                "created_at": api_key.created_at
            },
            "raw_token": raw_token
        })),
    ))
}
```
Note: `KeyService::create` returns `Result<_, DbError>`, not `Result<_, MnemonicError>`. Use `.map_err(|e| ApiError::Internal(crate::error::MnemonicError::Db(e)))` to map DbError through MnemonicError to ApiError.

**Add list_keys_handler (D-16):**
```rust
/// GET /keys — returns all key metadata, never including raw token or hashed_key.
async fn list_keys_handler(
    State(state): State<AppState>,
) -> Result<Json<serde_json::Value>, ApiError> {
    let keys = state.key_service.list().await
        .map_err(|e| ApiError::Internal(crate::error::MnemonicError::Db(e)))?;
    Ok(Json(serde_json::json!({
        "keys": keys.iter().map(|k| serde_json::json!({
            "id": k.id,
            "name": k.name,
            "display_id": k.display_id,
            "agent_id": k.agent_id,
            "created_at": k.created_at,
            "revoked_at": k.revoked_at
        })).collect::<Vec<_>>()
    })))
}
```

**Add revoke_key_handler (D-17):**
```rust
/// DELETE /keys/:id — revokes a key (soft delete). Returns confirmation.
async fn revoke_key_handler(
    State(state): State<AppState>,
    Path(id): Path<String>,
) -> Result<Json<serde_json::Value>, ApiError> {
    state.key_service.revoke(&id).await
        .map_err(|e| ApiError::Internal(crate::error::MnemonicError::Db(e)))?;
    Ok(Json(serde_json::json!({ "revoked": true, "id": id })))
}
```

**Test 1 — test_scope_mismatch_returns_403 (AUTH-04-a):**
Create scoped key for agent-A. POST /memories with agent_id: "agent-B" and Authorization: Bearer <token>. Assert response status is 403. Assert response body contains `"error": "forbidden"` and `"detail"` containing "agent-A" and "agent-B".

**Test 2 — test_scope_forces_agent_id (AUTH-04-b):**
Create scoped key for agent-A. POST /memories with content but NO agent_id, with Authorization: Bearer <token>. Assert 201. Then GET /memories with Authorization: Bearer <token>. Assert the returned memory has agent_id = "agent-A".

**Test 3 — test_wildcard_key_passes_through (AUTH-04-c):**
Create wildcard key (agent_id = None). POST /memories with agent_id: "any-agent", with Authorization: Bearer <token>. Assert 201. Response memory should have agent_id "any-agent".

**Test 4 — test_scoped_delete_wrong_owner_403 (AUTH-04-d):**
Create scoped key for agent-A. First create a memory with agent_id "agent-B" in open mode (before key exists) OR create via a wildcard key. Then DELETE /memories/{id} with the scoped agent-A key. Assert 403.
Strategy: Use build_test_state, create memory via service.create_memory with agent_id "agent-B" directly, then create scoped key, then DELETE via HTTP.

**Test 5 — test_scoped_delete_own_memory_ok (AUTH-04-e):**
Create scoped key for agent-A. Create a memory with agent_id "agent-A" via service.create_memory. Then DELETE /memories/{id} with the scoped key. Assert 200 (or the successful delete response).

**Test 6 — test_post_keys_creates_key (KEY-endpoint-a):**
In open mode (zero keys), POST /keys with body `{"name": "my-key"}`. Assert 201. Assert response has `raw_token` field starting with "mnk_". Assert response has `key` object with `id`, `name`, `display_id`, `created_at` fields. Assert `name` == "my-key".

**Test 7 — test_get_keys_no_raw_token (KEY-endpoint-b):**
Create a key via POST /keys in open mode. Then create a second key with auth. GET /keys with Authorization header. Assert 200. Assert response has `keys` array. Assert NO element in the array has a `raw_token` or `hashed_key` field. Assert at least one element has `name`, `id`, `display_id`, `created_at` fields.

**Test 8 — test_delete_key_revokes_access (KEY-endpoint-c):**
Create two keys via POST /keys (first in open mode, second with auth from first key). DELETE /keys/:id with the first key's token to revoke the second key. Then use the second key's token to GET /memories. Assert 401.

Important implementation notes:
- For tests that need both a key AND to make HTTP requests, build the Router from state after creating the key
- For tests that POST /keys in open mode, just make the request — middleware passes through with zero keys
- Use `json_request("POST", "/keys", json!({"name": "..."}))` for key creation requests
- Use the existing `response_json()` helper to parse response bodies
- For requests needing auth header, build requests manually with `.header("authorization", format!("Bearer {}", token))`
- The `build_router(state.clone())` call can be made multiple times with the same state for multiple oneshot requests
