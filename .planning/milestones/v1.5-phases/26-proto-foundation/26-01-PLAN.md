    package mnemonic.v1;

      rpc SearchMemories(SearchMemoriesRequest) returns (SearchMemoriesResponse);

    **Shared Memory message (D-05):**
    ```protobuf
    message Memory {
      string id = 1;
      string content = 2;
      string agent_id = 3;
      string session_id = 4;
      repeated string tags = 5;
      google.protobuf.Timestamp created_at = 6;
      string embedding_model = 7;
    }
    ```

    **RPC-specific request/response pairs (D-04):**

    StoreMemory — request has content (string), agent_id (string), session_id (string), tags (repeated string). Response wraps a Memory.

    SearchMemories — request has query (string), agent_id (string), session_id (string), tags (repeated string), limit (int32). Response has repeated SearchResult. SearchResult has Memory + float distance.

    ListMemories — request has agent_id (string), session_id (string), tags (repeated string), limit (int32), offset (int32). Response has repeated Memory memories + int32 total.

    DeleteMemory — request has id (string). Response wraps the deleted Memory.

    Field types per D-03: string for IDs, repeated string for tags, float for distances, google.protobuf.Timestamp for timestamps, int32 for limit/offset/total.

    Use proto3 defaults for optional fields (session_id = empty string, tags = empty list) rather than `optional` wrappers — this avoids `prost::Option<String>` complexity in generated Rust types. Per Claude's Discretion in CONTEXT.md.

    Create `build.rs` at project root with conditional proto codegen. Per D-07 and D-09:

    ```rust
    // build.rs
    fn main() {
        // Only run proto codegen when interface-grpc feature is active.
        // IMPORTANT: cfg!(feature = "...") does NOT work in build scripts.
        // Build scripts are compiled as standalone binaries — feature flags
        // propagate via CARGO_FEATURE_ environment variables, not cfg attributes.
        if std::env::var("CARGO_FEATURE_INTERFACE_GRPC").is_err() {
            return;
        }

        // Explicit rerun-if-changed BEFORE compile_protos.
        // Prevents always-dirty incremental build bug (tonic issue #2239):
        // compile_protos may emit cargo:rerun-if-changed with an unresolvable
        // path. Our explicit directive ensures Cargo tracks the real file.
        println!("cargo:rerun-if-changed=proto/mnemonic.proto");
        println!("cargo:rerun-if-changed=build.rs");

        // Full relative path form — NOT bare "mnemonic.proto" with include dir.
        // The literal argument becomes the rerun-if-changed path; "proto/mnemonic.proto"
        // exists on disk, bare "mnemonic.proto" does not.
        tonic_build::compile_protos("proto/mnemonic.proto")
            .expect("Failed to compile proto/mnemonic.proto");
    }
    ```

    Key anti-patterns to AVOID:
    - Do NOT use `cfg!(feature = "interface-grpc")` — does not work in build scripts (Pitfall 4)
    - Do NOT use `compile_protos("mnemonic.proto")` with a separate include dir — triggers always-dirty bug (Pitfall 2)
    - Do NOT omit the `println!("cargo:rerun-if-changed=...")` lines — belt-and-suspenders against issue #2239
