### Integration Finding: CI Release Feature Flag

The CI release workflow (`.github/workflows/release.yml`) installs protoc (PROTO-03) but does not pass `--features interface-grpc` to `cargo build --release`. Released binaries ship as REST-only with no gRPC capability.

This is consistent with the feature-gated design (PROTO-04) — gRPC is opt-in. If gRPC should ship in release binaries, add `--features interface-grpc` to the CI build command. If REST-only releases are intentional, this is working as designed.

### Minor: Reflection v1alpha Bypass

The auth bypass in `src/grpc/auth.rs:85` only covers `/grpc.reflection.v1.ServerReflection/`. Clients using the older v1alpha reflection API would receive `Unauthenticated`. This was an explicit design choice per RESEARCH.md.
