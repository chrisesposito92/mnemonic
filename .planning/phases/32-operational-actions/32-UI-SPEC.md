> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Interaction Contracts

### Tab Bar (D-01)
- Add fourth tab: `{ id: 'compact', label: 'Compact', href: '#/compact' }`
- Tab type union becomes: `'memories' | 'agents' | 'search' | 'compact'`
- `parseHash()` adds: `if (hash === '#/compact') return 'compact'`
- All existing tab behavior unchanged — active tab accent underline, inactive muted text

### CompactTab State Machine (D-11)

Primary visual anchor — idle state: "Run Dry Run" button (accent-colored, rightmost control in the controls row). Primary visual anchor — preview state: summary line ("N clusters found, M memories -> K compacted") at 14px/600, positioned above the cluster table.

```
idle ──[mount]──> idle (show controls, no results)
idle ──[Run Dry Run]──> loading-dry-run
loading-dry-run ──[API success, clusters > 0]──> preview
loading-dry-run ──[API success, clusters === 0]──> empty
loading-dry-run ──[API error]──> error
loading-dry-run ──[401/403]──> onUnauthorized (exits tab)
preview ──[Confirm Compact]──> loading-execute
preview ──[Discard Preview]──> idle (reset, keep agent/threshold)
loading-execute ──[API success]──> success
loading-execute ──[API error]──> error
loading-execute ──[401/403]──> onUnauthorized (exits tab)
success ──[user interaction]──> idle (controls reset for next run)
error ──[user interaction]──> idle (retry available)
```

### CompactTab Controls (D-02, D-03)
- **Agent scope dropdown:** `<select>` populated from `fetchStats()` on mount. Same pattern as MemoriesTab/SearchTab agent dropdown. Required field — cannot run dry-run without agent selection.
- **Similarity threshold input:** `<input type="number">` with step="0.01", min="0", max="1", default value "0.85". Body-size text (14px/400). Label "threshold:" in muted label style (12px/400).
- **max_candidates:** Not exposed in UI (D-03). Stays at API default (100).
- **Run Dry Run button:** Accent background, dark text, 14px/600. Disabled when no agent selected or when loading. Same style as "Search Memories" button.
- Layout: horizontal row with `gap: 16px`, `alignItems: center`, `flexWrap: wrap`.

### Dry-Run Flow (D-04, D-06, D-07, D-08)
1. User selects agent, optionally adjusts threshold, clicks "Run Dry Run"
2. `POST /memories/compact` with `{ agent_id, threshold, dry_run: true }`
3. Loading state: SkeletonRows with `rows={5}`
4. On success: parse `CompactResponse` — extract `clusters_found`, `memories_merged`, `memories_created`, `id_mapping`
5. If `clusters_found === 0`: show empty state
6. If `clusters_found > 0`: show preview state with summary line + cluster table

### Dry-Run Diff Display (D-06, D-07, D-08)
- **Summary line:** "N clusters found, M memories -> K compacted" at 14px/600 (emphasized body). Positioned above cluster table with `marginBottom: 16px`.
- **Cluster table:** One section per cluster from `id_mapping`. Each cluster renders as:
  - Cluster header: "Cluster {n}" at 12px/400/muted
  - Source memories: fetched by ID from `GET /memories/{id}`. Each source shown with tree prefix:
    - All but last: `"  \u251C "` (box-drawing tee) + truncated content (80 chars)
    - Last in group: `"  \u2514 "` (box-drawing corner) + truncated content (80 chars)
  - Content preview: 14px/400/text-primary
  - Tree prefixes: 12px/400/text-muted
- **Cluster group styling:** Each cluster group has `padding: 8px 0`, separated by `borderBottom: 1px solid var(--color-border)`
- **Source memory fetch:** For each `source_ids` array in `id_mapping`, fetch `GET /memories/{id}` individually. Display "Loading..." placeholder (12px/400/muted) while fetching. If a fetch fails, show memory ID instead of content.

### Confirm/Discard (D-09)
- **Confirm Compact button:** Accent background, dark text, 14px/600. Label "Confirm Compact". Positioned below cluster table with `marginTop: 16px`.
- **Discard Preview button:** Transparent background, border `1px solid var(--color-border)`, 14px/400, text-muted. Label "Discard Preview". Adjacent to Confirm with `gap: 8px`.
- Confirm calls `POST /memories/compact` with `{ agent_id, threshold, dry_run: false }` (same params minus dry_run flag).
- Discard Preview returns to idle state, preserving agent and threshold selections.

### Post-Compaction Success State
- Summary text: "Compaction complete. N clusters merged M memories into K." at 14px/400/text-primary.
- Below summary: "Run another compaction" link-style text at 14px/400/accent, `cursor: pointer`, `textDecoration: underline`. Clicking resets to idle state.

### Empty State (clusters_found === 0)
- Centered text block at `padding: 48px 0` (matches other tabs)
- Heading: "No compactable clusters found" at 14px/400/text-primary
- Body: "All memories for this agent are already distinct enough at the current threshold. Try lowering the threshold for more aggressive clustering." at 12px/400/text-muted, `marginTop: 8px`

### Error State
- Uses existing `ErrorMessage` component
- Copy: "Compaction failed. Check server logs and try again." (same pattern as other tabs)

### Cross-Tab Refresh (D-05)
- After compaction completes (success state), other tabs auto-refresh on next navigation. No cross-tab state synchronization needed — each tab already fetches data on mount via useEffect. No additional implementation required.

### Existing Tab Audit (D-10, D-12)
- **MemoriesTab:** Already has loading/loaded/empty/error states. No gaps found.
- **AgentsTab:** Already has loading/loaded/empty/error states. No gaps found.
- **SearchTab:** Has idle/loading/loaded/empty/error states. No gaps found.
- **Header:** Has loading/loaded/error states. No gaps found.
- All tabs use per-tab error handling with inline ErrorMessage (D-12). No global error boundary.

## API Contract

New typed wrappers to add to `dashboard/src/api.ts`:

### `compactMemories(token, params, signal)`

```typescript
interface CompactParams {
  agent_id: string
  threshold?: number
  dry_run?: boolean
}

interface ClusterMapping {
  source_ids: string[]
  new_id: string | null
}

interface CompactResponse {
  run_id: string
  clusters_found: number
  memories_merged: number
  memories_created: number
  id_mapping: ClusterMapping[]
  truncated: boolean
}
```

Calls `POST /memories/compact` with JSON body. Uses `apiFetch` with `method: 'POST'`, `Content-Type: application/json`.

### `fetchMemoryById(token, id, signal)`

```typescript
// Returns: Memory (same type as in ListResponse)
```

Calls `GET /memories/{id}`. Uses existing `apiFetch` wrapper. Returns single `Memory` object.
