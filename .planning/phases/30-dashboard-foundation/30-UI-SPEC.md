# Phase 30 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 30 delivers a proof-of-life app shell only. The heading, health card, and loading/error states defined here are the entire UI surface. Phase 31 replaces this shell with the full dashboard. Design decisions made here establish the base tokens Phase 31 inherits.

## Typography

All text uses the system monospace stack declared in the Design System section.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Health card field values, status text |
| Label | 12px | 400 (regular) | 1.4 | Field keys (e.g. "backend:", "status:") |
| Heading | 20px | 600 (semibold) | 1.2 | Page heading: "Mnemonic Dashboard" |
| Display | 28px | 600 (semibold) | 1.2 | Not used in Phase 30 — reserved for Phase 31 |

**Constraints:**
- Exactly 2 weights used: 400 and 600.
- Exactly 3 sizes active in Phase 30: 12px, 14px, 20px.
- No italic. No underline except on links.

## Component Inventory

Phase 30 has exactly three visual components. All are hand-written Preact functional components — no third-party component library.

### 1. `AppShell` (`dashboard/src/App.tsx`)

Full-viewport dark surface. Centers `HealthCard` vertically and horizontally. Renders the page heading above the card.

```
Layout: min-h-screen flex flex-col items-center justify-center gap-xl (32px)
Background: var(--color-bg)
```

### 2. `HealthCard` (`dashboard/src/components/HealthCard.tsx`)

Rounded card showing `GET /health` response fields. Three states: loading, loaded, error.

```
Width: max-w-sm w-full
Background: var(--color-surface)
Border: 1px solid var(--color-border)
Border radius: 8px (2px grid exception — standard UI affordance)
Padding: 24px (lg)
```

Field rows: label (12px muted) + value (14px text-primary), 8px gap between rows.

### 3. Inline status dot

12×12px circle rendered inside `HealthCard` next to the `status:` field value.

```
ok state:    background var(--color-accent)   (#22d3ee)
error state: background var(--color-error)    (#ef4444)
```

## States and Interactions

### HealthCard loading state

- Renders three placeholder rows at full width using `background: var(--color-border)` at 40% opacity
- Height: 12px per row, 8px gap between rows
- No spinner — static skeleton only (Phase 31 will introduce animated skeletons)
- Copy: none (no text in loading state)

### HealthCard loaded state

Displays these fields from the `GET /health` response:

| Field label | Field value source |
|-------------|-------------------|
| `status:` | `response.status` + status dot |
| `backend:` | `response.backend` |
| `version:` | `response.version` (if present) or `--` |

### HealthCard error state

- Replaces field rows with a single error message line
- Text: 14px, `var(--color-error)`, weight 400
- Copy: see Copywriting Contract below

### No interactive elements in Phase 30

There are no buttons, links, forms, or user-triggerable actions. The only interaction is the automatic `GET /health` fetch on component mount. No hover states, focus states, or click handlers required.

## Vite / Build Configuration Contract

These constraints govern what the build pipeline produces and are part of the visual contract because they determine what gets embedded.

| Constraint | Value | Source |
|------------|-------|--------|
| Output target | Single `index.html` with inlined JS + CSS (via vite-plugin-singlefile) | CONTEXT.md D-08 |
| Fallback | Multi-file output in `dashboard/dist/` with axum-embed asset routing | CONTEXT.md D-09 |
| Routing strategy | Hash routing (`#/path`) — no history API | CONTEXT.md D-14 |
| Dev server port | `:5173` with API proxy to `:8080` | CONTEXT.md D-07 |
| Build output dir | `dashboard/dist/` | default for Vite |

The app shell must render correctly when served from any path under `/ui/` because the Rust side serves `/ui/*` via axum-embed and SPA fallback returns `index.html` for all non-asset paths.
