  <action>
Create the directory `.github/workflows/` and the file `release.yml`.

The workflow must:

1. **Trigger** on push to tags matching `v*` pattern.

2. **Build job** — matrix strategy with three targets:

| name | os | target | artifact name |
|------|-----|--------|---------------|
| linux-x86_64 | ubuntu-latest | x86_64-unknown-linux-gnu | mnemonic-linux-x86_64 |
| macos-x86_64 | macos-latest | x86_64-apple-darwin | mnemonic-macos-x86_64 |
| macos-aarch64 | macos-latest | aarch64-apple-darwin | mnemonic-macos-aarch64 |

Build job steps:
- `actions/checkout@v4`
- `dtolnay/rust-toolchain@stable` with `targets: ${{ matrix.target }}` (CRITICAL: must add target for macOS Intel cross-compile on ARM runner)
- `cargo build --release --target ${{ matrix.target }}`
- Stage binary: `mkdir -p dist && cp target/${{ matrix.target }}/release/mnemonic dist/${{ matrix.artifact }} && tar -czf dist/${{ matrix.artifact }}.tar.gz -C dist ${{ matrix.artifact }}`
- `actions/upload-artifact@v4` with `name: ${{ matrix.artifact }}` and `path: dist/${{ matrix.artifact }}.tar.gz`

3. **Release job** — runs after all build jobs complete:
- `needs: build`
- `runs-on: ubuntu-latest`
- `permissions: contents: write` (required for creating releases)
- `actions/download-artifact@v4` (no arguments — downloads all artifacts)
- `softprops/action-gh-release@v2` with `files: '**/*.tar.gz'`

4. **Action versions** (verified current as of research):
- `actions/checkout@v4`
- `dtolnay/rust-toolchain@stable` (not `actions-rs/toolchain` which is deprecated)
- `actions/upload-artifact@v4` (not v3 which is deprecated)
- `actions/download-artifact@v4`
- `softprops/action-gh-release@v2`

Do NOT use the `cross` tool — all three targets build natively on their respective runners (macOS runner supports both aarch64 native and x86_64 cross-compile with the target added).

Do NOT include Docker image building — this is deferred per CONTEXT.md.

The workflow name should be `Release`.
  </action>
