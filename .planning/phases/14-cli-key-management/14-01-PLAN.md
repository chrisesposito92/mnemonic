**src/auth.rs:** Add a new method `find_by_display_id` to the `impl KeyService` block, after the existing `revoke` method and before `validate`. This method queries the `api_keys` table by `display_id` column and returns all matching rows (0, 1, or more). This is needed because `KeyService::revoke()` takes a full UUID, but the CLI accepts 8-char display_id prefixes.

Implementation:
```rust
/// Finds API keys by their 8-char display_id prefix.
/// Returns all matches — caller handles 0 (not found), 1 (exact), or >1 (ambiguous).
pub async fn find_by_display_id(&self, display_id: &str) -> Result<Vec<ApiKey>, crate::error::DbError> {
    let did = display_id.to_string();
    self.conn
        .call(move |c| -> Result<Vec<ApiKey>, rusqlite::Error> {
            let mut stmt = c.prepare(
                "SELECT id, name, display_id, agent_id, created_at, revoked_at FROM api_keys WHERE display_id = ?1"
            )?;
            let rows = stmt.query_map(rusqlite::params![did], |row| {
                Ok(ApiKey {
                    id: row.get(0)?,
                    name: row.get(1)?,
                    display_id: row.get(2)?,
                    agent_id: row.get(3)?,
                    created_at: row.get(4)?,
                    revoked_at: row.get(5)?,
                })
            })?;
            rows.collect::<Result<Vec<_>, _>>()
        })
        .await
        .map_err(crate::error::DbError::from)
}
```

2. **`run_keys` dispatcher function** -- takes `KeysSubcommand` and `KeyService`, dispatches to the correct handler. This is the single entry point called from main.rs:
```rust
pub async fn run_keys(subcommand: KeysSubcommand, key_service: crate::auth::KeyService) {
    match subcommand {
        KeysSubcommand::Create { name, agent_id } => cmd_create(key_service, name, agent_id).await,
        KeysSubcommand::List => cmd_list(key_service).await,
        KeysSubcommand::Revoke { id } => cmd_revoke(key_service, id).await,
    }
}
```

3. **`cmd_create` function** (per D-08, D-09, D-10, D-19, D-20):
   - Calls `key_service.create(name, agent_id).await`
   - On success: prints raw token on its own line to stdout (first line, for easy piping), then prints key metadata (ID, Name, Scope) to stdout, then prints "Save this key -- it will not be shown again." warning to stderr via `eprintln!`
   - On error: prints error to stderr via `eprintln!` and calls `std::process::exit(1)`
   - Output format:
     ```
     {raw_token}               <- stdout, line 1 (pipeable)
     ID:    {display_id}       <- stdout
     Name:  {name}             <- stdout
     Scope: {agent_id or "(unscoped)"}  <- stdout
                               <- stderr blank line
     Save this key -- it will not be shown again.  <- stderr
     ```

4. **`cmd_list` function** (per D-11, D-12, D-13, D-14, D-19, D-20):
   - Calls `key_service.list().await`
   - If empty: prints "No API keys found. Create one with: mnemonic keys create <name>" to stdout and returns
   - If keys exist: prints formatted table with header row and separator line
   - Column format: `{:<10}  {:<20}  {:<20}  {:<19}  {}` for ID, NAME, SCOPE, CREATED, STATUS
   - STATUS column: "active" if `revoked_at` is None, otherwise `format!("revoked ({})", &revoked_at[..10])` showing date only
   - Truncate name to 20 chars (17 + "...") if longer than 20
   - Truncate scope to 20 chars (17 + "...") if longer than 20
   - Scope displays `(all)` when agent_id is None
   - Created column shows first 19 chars of created_at string (datetime without timezone)
   - On error: prints error to stderr and calls `std::process::exit(1)`

5. **`cmd_revoke` function** (per D-15, D-16, D-17, D-18, D-19, D-20):
   - Helper function `fn is_display_id(input: &str) -> bool`: returns true if `input.len() == 8 && input.chars().all(|c| c.is_ascii_hexdigit())`
   - If input is a display_id (8 hex chars):
     - Calls `key_service.find_by_display_id(&id).await`
     - 0 results: `eprintln!("No key found with ID {id}")` then `std::process::exit(1)`
     - 1 result: extract the full UUID from `found[0].id`, then call `key_service.revoke(&full_id).await`, then `println!("Key {} revoked", found[0].display_id)`
     - >1 results: `eprintln!("Ambiguous -- {} keys match prefix. Use full UUID:", found.len())`, then list each match with `eprintln!("  {} ({})", key.id, key.name)`, then `std::process::exit(1)`
   - If input is NOT a display_id (assumed full UUID):
     - Calls `key_service.revoke(&id).await` directly
     - On success: `println!("Key revoked")` (no ID echoed -- the UUID is what the user typed, and we skip an extra DB lookup to retrieve display_id)
     - On error: prints error to stderr via `eprintln!` and calls `std::process::exit(1)`
