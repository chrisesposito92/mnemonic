| Border | --color-border | #2a2a2a | Table dividers, card borders, input borders, skeleton bars |
| Text primary | --color-text | #e5e5e5 | Body text, memory content, metadata values |
| Text muted | --color-text-muted | #6b7280 | Labels, column headers, placeholder text, pagination text |

Accent reserved for:
1. Health indicator dot when status = "ok" (green-equivalent signal)
2. Active tab underline indicator
3. Distance score bar fill (similarity visualization)
4. "Connect" button background on login screen

Destructive reserved for:
1. Health indicator dot when status != "ok"
2. Inline error message text on login screen ("Invalid API key")
3. API error state messages in table panels

## Interaction Contracts

### Tab Bar (D-01, D-04)
- Hash routes: `#/memories`, `#/agents`, `#/search`
- Default route: `#/memories` (first tab active on load)
- Active tab: accent-colored bottom border (2px solid var(--color-accent)), text at full opacity
- Inactive tab: --color-text-muted, no border
- Tab switch: immediate (no animation); re-fetches tab data if stale
- Router: manual `useState` + `hashchange` event listener in App.tsx

### Memory Table (D-05, D-06, D-07, D-08)
- Columns: content preview (truncated to 80 characters + "..."), agent_id, session_id, tags (comma-separated), created_at (relative time, e.g., "2m ago")
- Row click: expands inline below row — shows full content, id, embedding_model, created_at (ISO full), updated_at; click again collapses
- Expanded row background: --color-surface with 1px border-top/bottom in --color-border
- Pagination: offset-based; shows "Showing 1–20 of 347"; Prev disabled on first page, Next disabled on last page
- Filter controls: three dropdowns (agent_id, session_id, tag); populated from unique values in current list response; changes reset offset to 0

### Search Tab (D-09, D-10, D-11)
- Query input: full-width text field, placeholder "Search memories..."
- Search triggers on Enter keypress or explicit Search button click (not on-type)
- Results table columns: distance bar, distance score (4 decimal places), content preview (80 chars), agent_id, session_id
- Distance bar: filled proportional to similarity score (1.0 = full width); bar background --color-accent at 30% opacity, fill --color-accent
- Optional filters: agent_id and tag dropdowns (same pattern as Memories tab)

### Login Screen (D-12, D-13, D-14, D-15)
- Triggers: 401 response on initial GET /health probe at mount
- Layout: full-screen centered column, --color-bg background
- Elements: "Mnemonic" heading (20px/600), "API key required" label (14px/400/muted), token input (14px/400 monospace, full-width max 400px), "Connect" button (accent background, --color-bg text, 14px/600)
- Token input: type="password", placeholder "mnk_...", no autocomplete
- Connect: fires GET /health with Authorization header; on 200 stores token in component state, renders dashboard; on 401 shows inline error "Invalid API key. Check your key and try again." below input in --color-error; clears input value
- Token never written to localStorage, sessionStorage, or any Web Storage API

### Header (D-02)
- Always visible above tab bar
- Health dot: 12x12px circle, --color-accent when ok, --color-error when not ok, --color-border when loading
- Backend name: 12px/400/muted, e.g., "sqlite" or "qdrant"
- Refreshes on a 30-second polling interval using the same fetch + AbortSignal.timeout pattern as HealthCard

### Skeleton Loading Pattern (extends HealthCard)
- 3 horizontal bars: height 12px, background --color-border, opacity 0.4, borderRadius 2px
- Gap between bars: 8px
- Appears immediately when fetch is in-flight; replaced by content on success or error message on failure
