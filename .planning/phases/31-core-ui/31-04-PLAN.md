**2. Create dashboard/src/components/SearchTab.tsx** (per D-09, D-10; BROWSE-03):

Props: `{ token: string | null }`

State:
- `query: string` — controlled input value, starts as ''
- `agentFilter: string` — optional agent_id filter, starts as ''
- `tagFilter: string` — optional tag filter, starts as ''
- `state: 'idle' | 'loading' | 'loaded' | 'error'` with associated data
- `results: SearchResult[]` where `SearchResult = { id, content, agent_id, session_id, tags, embedding_model, created_at, updated_at, distance }`
- `error: string | null`

Search is triggered ONLY on:
1. Enter keypress in the query input
2. Click on "Search Memories" button
NOT on every keystroke (per D-09, Research anti-pattern).

Search handler:
1. If query is empty/whitespace, return early (don't call API)
2. Set state to 'loading'
3. Build query params: `?q=${encodeURIComponent(query.trim())}` + `&agent_id=${agentFilter}` if non-empty + `&tag=${tagFilter}` if non-empty
4. Call `apiFetch(\`/memories/search?${params}\`, token)`
5. Parse JSON. Set results from `response.memories`. Set state to 'loaded'.
6. On error: set state 'error', error message "Failed to load data. Reload the page or check server status."

Layout:
- Search controls section (top):
  - Query input: full-width, fontSize 14px, fontWeight 400, fontFamily monospace, padding 8px 12px, background var(--color-surface), border 1px solid var(--color-border), borderRadius 4px, color var(--color-text), placeholder "Search memories..." (exact Copywriting Contract)
  - Below input, a row with optional filter dropdowns + search button:
    - agent_id select dropdown (label "agent_id:", same style as FilterBar) — populated as a simple text input or select. Since search doesn't return a pre-populated list, use a text input for agent_id filter instead of a select. fontSize 12px, same styling as FilterBar dropdowns.
    - tag text input (label "tag:") — same styling
    - "Search Memories" button (exact Copywriting Contract): padding 8px 16px, background var(--color-accent), color var(--color-bg), fontSize 14px, fontWeight 600, border none, borderRadius 4px, cursor pointer. Disabled while state === 'loading'.
  - Gap between search controls and results: 24px

- Results section:
  - Idle state (before first search): nothing shown
  - Loading state: SkeletonRows
  - Error state: ErrorMessage
  - Empty state (loaded but results.length === 0): centered heading "Nothing matched your search" and body "No memories matched your query. Try different search terms or broaden your filters." (exact Copywriting Contract)
  - Loaded with results: table with column headers then result rows

- Results table column headers: display grid, gridTemplateColumns 'auto auto 1fr auto auto', gap 16px, padding 8px 0, borderBottom 1px solid var(--color-border). Labels: "score", "distance", "content", "agent", "session" — fontSize 12px, color var(--color-text-muted)

- Each result row: display grid, same gridTemplateColumns, gap 16px, padding 8px 0, borderBottom 1px solid var(--color-border), alignItems center
  - DistanceBar component (spans first two columns or use separate cells): render `<DistanceBar distance={result.distance} />`
  - Content preview: `truncate(result.content, 80)`, fontSize 14px, color var(--color-text)
  - agent_id: fontSize 12px, color var(--color-text-muted). Display `(none)` if empty string.
  - session_id: fontSize 12px, color var(--color-text-muted). Display `(none)` if empty string.

Handle `onKeyDown` on the query input: if `e.key === 'Enter'`, trigger search.
