**4. Helper function to build the prompt data block** (file-private, used by OpenAiSummarizer and tests):
```rust
fn build_data_block(texts: &[String]) -> String {
    let mut block = String::from("<memories>\n");
    for (i, text) in texts.iter().enumerate() {
        block.push_str(&format!("<memory index=\"{}\">{}</memory>\n", i, text));
    }
    block.push_str("</memories>");
    block
}
```
The system message is a constant string: `"You are a memory consolidation assistant. Consolidate the provided memories into a single concise summary that preserves all important facts. Output only the summary text."`

**5. SummarizationEngine impl for OpenAiSummarizer:**
- Guard: if `texts.is_empty()`, return `Err(LlmError::ApiCall("cannot summarize empty cluster".into()))`
- Build system message (instruction-only, no user data) and user message (data block from `build_data_block`)
- Construct `ChatRequest` with `model: &self.model`, the two messages, and `temperature: 0.3`
- POST to `format!("{}/chat/completions", self.base_url)` with `.bearer_auth(&self.api_key).json(&req)`
- Error mapping chain (CRITICAL — use `e.is_timeout()` check):
  - `.send().await.map_err(|e| if e.is_timeout() { LlmError::Timeout } else { LlmError::ApiCall(e.to_string()) })?`
  - `.error_for_status().map_err(|e| LlmError::ApiCall(e.to_string()))?`
  - `.json::<ChatResponse>().await.map_err(|e| LlmError::ParseError(e.to_string()))?`
- Extract summary: `resp.choices.into_iter().next().map(|c| c.message.content).ok_or_else(|| LlmError::ParseError("empty choices array".into()))?`
- Optional: add `tracing::debug!(model = %self.model, texts_count = texts.len(), "sending summarization request")` before the POST
