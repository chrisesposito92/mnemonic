### E2E Flows Verified

1. **Store memory** — POST /memories → embed → atomic dual-table insert → 201 + Memory
2. **Semantic search** — GET /memories/search → embed query → CTE KNN + filters → ranked results
3. **List memories** — GET /memories → COUNT + SELECT with filters → paginated list
4. **Delete memory** — DELETE /memories/:id → fetch → atomic dual-table delete → 200 or 404
5. **Startup chain** — register_sqlite_vec → init_tracing → load_config → db::open → engine init → MemoryService → serve

### Integration Issue (Non-Critical)

**`embedding_provider` config field is inert** — `src/main.rs` selects the embedding provider based solely on `config.openai_api_key.is_some()`, ignoring the `config.embedding_provider` value. The README documents `MNEMONIC_EMBEDDING_PROVIDER` as configurable, but setting it has no effect beyond logging. This is a documentation/behavior mismatch, not a runtime failure. The actual requirement (EMBD-04: "set OPENAI_API_KEY to use OpenAI") works correctly.
